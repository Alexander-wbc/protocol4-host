# Protocol 4 单窗口 UDP 演示平台

一个基于 Python 的 **协议四（1 比特滑动窗口协议）** 图形化演示程序。  
程序采用 **单窗口界面**，但内部同时运行两个真实的 **UDP 节点 A / B**，用于在一台电脑上完成协议四的发送、接收、确认、超时重传与错误处理演示。

## 项目简介

本项目使用 Python `tkinter` 编写图形界面，使用 `socket` 实现 UDP 通信，内部在本机回环地址上创建两个节点：

- 节点 A：`127.0.0.1:9000`
- 节点 B：`127.0.0.1:9001`

两个节点之间通过真实 UDP socket 通信，但所有过程都集中显示在同一个窗口中，便于课程展示与实验答辩。

本项目实现了协议四的核心机制：

- 1 bit 序号 `0/1`
- 停等发送
- ACK 确认
- 超时重传
- 重复帧识别
- CRC32 校验
- 确定性错误注入演示

## 协议四简介

协议四可以看作一种 **1 比特滑动窗口协议**，本质上属于停等式可靠传输协议。

它的基本思想是：

1. 发送方每次只发送一个数据帧
2. 发送后等待接收方返回 ACK
3. 若在规定时间内未收到 ACK，则超时重传
4. 接收方根据帧序号判断该帧是否是当前期望的新帧
5. 若收到重复帧，则不重复交付上层，只重发 ACK

由于序号空间只有 1 bit，因此帧序号仅有两种取值：

- `0`
- `1`

## 主要功能

### 1. 单窗口双节点显示
程序只打开一个 GUI 窗口，但内部同时运行两个协议四节点：

- A 节点
- B 节点

支持：

- A → B 发送消息
- B → A 发送消息

### 2. 核心状态显示
每个节点界面中只保留三个核心状态：

- `send_seq`：下一次发送的数据帧序号
- `expected_seq`：当前期望接收的数据帧序号
- `ACK状态灯`
  - 绿色：空闲
  - 红色：正在等待 ACK

### 3. 总日志显示
日志窗口会显示完整协议过程，例如：

- DATA 发送
- ACK 发送
- DATA 接收
- ACK 接收
- 校验失败
- 超时重传
- 重复帧识别
- 错误注入信息

### 4. 错误演示模式
本项目支持几种确定性错误注入模式，便于稳定演示协议四的错误恢复能力：

- `normal`
- `drop_first_data`
- `drop_first_ack`
- `corrupt_first_data`
- `corrupt_first_ack`
